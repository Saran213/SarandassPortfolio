Querying Largest Database

This project focuses on querying a large relational database using SQL to derive actionable insights from customer and sales data. Using the AdventureWorks sample database, participants will restore the database on a server and perform a series of SQL queries to extract information such as customer details, sales performance for specific time periods, and cumulative sales trends. Key topics include understanding table structures, data types, SQL operators, and functions. The tasks involve retrieving detailed customer contact information, analyzing sales data from specified timeframes, and aggregating total sales by customers and months. This project provides a hands-on approach to mastering SQL querying techniques in a real-world database environment, enhancing skills essential for database management and analytics.

Relational Database Design

This project aims to provide hands-on experience in designing and implementing a relational database using SQL Server. Participants will learn to transform a relational database design into well-structured SQL tables, incorporating attributes, primary keys, and foreign keys to ensure data integrity and proper relationships between entities. The project involves defining relational attributes, creating primary and foreign keys, inserting sample data into each table, and performing data deletion to understand database operations. By completing this project, participants will gain foundational skills in database design, normalization, and implementation, essential for managing relational databases effectively.

Case Study

This case study explores the use of SQL queries to extract valuable insights from a dataset containing information about sales, profit, marketing expenditure, and other relevant factors. The dataset comprises three key tables: FactTable, ProductTable, and LocationTable. The FactTable contains detailed transaction data, while the ProductTable and LocationTable provide additional context about products and geographic locations, respectively. By leveraging SQL queries, the database administrator aims to uncover patterns and trends in the data, such as identifying top-selling items, analyzing state-wise sales and profit, and assessing the impact of marketing spending. The insights derived from these queries can be instrumental in making informed business decisions and optimizing sales strategies.
